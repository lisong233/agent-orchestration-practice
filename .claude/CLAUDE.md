# AI 军团 — Agent 编排作业

> 基于训练数据，构建多 Agent 编排系统：读取业务文档 → 发现隐含判断规则 → 输出审核结果 + 命中规则 + 判断依据

## docs 目录规范

**两层分工**（按文件产生时机和用途，不是按作者）：

| 目录 | 前缀 | 含义 | 产生时机 | 给谁看 |
|------|------|------|---------|--------|
| `docs/spec/` | **S** | Spec = 施工蓝图 | 高级模型独立审题后 | 执行模型 |
| `docs/status/` | **R** | Report = 执行报告 | 执行模型跑完一轮后 | 高级模型 review |

**命名规则**：`{S|R}{阶段号}-{日期}-{简短主题}.md`
- 阶段号对齐：S02 执行后产出 R02，S03 执行后产出 R03
- 附属文件：S02a / S03a 等
- 日期格式：YYYY-MM-DD

**当前文档链条**：

```
S01 初探 ──→ S02 施工蓝图 + S02a 执行附录 ──执行──→ R02 工程现状
                                                          │
                                              ┌── review ──┘
                                              ↓
                                           S03 规则重做 ──执行──→ R03 数据真相
                                                                      │
                                                          ┌── review ──┘
                                                          ↓
                                                       S04 harness收尾 ──执行──→ R04 执行发现
                                                                                  │
                                                                      ┌── review ──┘
                                                                      ↓
                                                                   (等待 S05)
```

**铁律**：
- spec 写"做什么"和"怎么做"——决策已做完，不留白
- status 写"发现了什么"和"建议高级模型考虑什么"——**不替高级模型做决策**
- status 里的施工建议必须由高级模型审阅后，自己写成下一份 spec
