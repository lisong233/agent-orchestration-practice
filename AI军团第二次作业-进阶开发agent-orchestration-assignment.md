# 智能体编排业务判断挑战

你需要提交一个可运行的智能体系统。系统读取业务文档，判断某个条件是否通过，例如“本项目创新程度是否足够”，“本项目是否可以通过立项申请”

公开文档只用于说明文档类型、业务场景和数据结构。公开文档上的跑分效果不计入最终成绩。

正式评分只在隐藏数据集上进行。本任务实际上是利用模型的世界知识，尽可能在最终硬标签的信号下发掘软标签（过程标签）的语义。

Hints：
1. 公开数据集的使用方式，是否应该一次性暴露给智能体？
2. llm会天生倾向以过拟合的方式满足硬标签的正确率
3. 用户intent能否准确触发你的工作流（trigger）
4. 本数据集是高度imbalance的，原因是什么？
5. 可能会用到的agent相关的能力：skill trigger、spec-handoff、compact方法、长期记忆管理、research loop

## 任务

你需要做一个可公网访问的网页系统。评委会在网页上上传业务文档，并输入一个判断 intent。你的系统需要返回：

- 每条数据的硬标签：`通过` / `不通过`
- 命中的判断规则（多条）
- 简短判断依据

公开数据只用于训练和理解任务。最终评测数据不会对外暴露。

## 数据

公开数据包含两类：

1. `计划任务书`
2. `立项申请书`

公开数据会直接给出标签数据（通过/不通过）。每类数据集约有 `XX` 条判断规则，具体规则以及数量不会公开。

公开数据用于帮助你理解：

- 文档类型
- 字段结构
- 业务语境
- 标签口径
- 规则范围

公开数据上的效果不直接计入最终成绩。

## 提交物

评测 DDL 前，你需要提交：

```text
公网网页 URL
README.md
design.md
源代码仓库或 zip
```

`design.md` 需要说明：

- 系统如何处理两类数据集
- 智能体或模块如何分工
- 如何定位判断规则
- 如何输出硬标签
- 如何保证新文档上的泛化能力

## 网页要求

评委打开你的公网网页后，必须能完成以下流程：

1. 选择数据集类型：`计划任务书` 或 `立项申请书`
2. 上传最终评测数据
3. 输入 intent，例如“判断创新程度是否足够”
4. 点击运行
5. 页面返回每条数据的判断结果

返回结果至少包含：

```json
{
  "id": "sample_001",
  "dataset_type": "计划任务书",
  "intent": "判断创新程度是否足够",
  "label": "通过",
  "matched_rules": [
    {
      "rule_id": "R-03",
      "rule_name": "技术路线具备实质创新",
      "evidence": "文档中对应依据"
    }
  ],
  "reason": "简短判断说明"
}
```

## 评分

最终分数由三部分组成：

1. **方案与工程质量**  
   看方案设计是否合理，代码是否可读、可维护、可扩展。

2. **硬标签评分**  
   在隐藏测试集上比较 `通过 / 不通过` 标签是否正确。

3. **规则语义对齐**  
   检查系统是否定位到正确的判断规则，例如某条 `XXX` 规则是否被识别并用于判断。

最终评测只使用隐藏数据集。隐藏数据集不会提前提供给参赛者。

## 禁止行为

- 只记忆公开样例，不能处理新文档
- 提交无法公网访问或无法复现的系统

## 核心要求

本作业考察的不是单次问答，而是：

> 设计一个能理解业务文档、输出硬标签、定位判断规则，并迁移到隐藏数据集的智能体系统。
